## 3-4. 오류를 디버깅하는 방법

~~~
✅ 학습 목표 :
* 오류의 정의에 대해 설명할 수 있다. 
* 오류 메시지를 보고 디버깅이라는 과정을 수행할 수 있다. 
~~~

<대표적인 SQL 오류 유형: 문법 오류 (Syntax Error)

문법 오류는 SQL 문법 규칙을 지키지 않았을 때 발생한다. 이러한 오류를 해결하기 위해서는 에러 메시지를 단순히 넘기지 말고, 내용을 해석하고 원인을 파악하는 것이 중요하다.

첫 번째로, SELECT 목록이 비어 있을 때 발생하는 오류이다.
"SELECT list must not be empty"라는 메시지는 SELECT 뒤에 가져올 컬럼이 없다는 의미이다.
예를 들어, SELECT만 작성하고 컬럼을 쓰지 않으면 오류가 발생한다.
이 경우 SELECT 뒤에 반드시 하나 이상의 컬럼을 작성해야 한다.

두 번째는 COUNT 함수 인자 개수 오류이다.
"Number of arguments does not match for aggregate function COUNT"라는 메시지는 COUNT 함수에 잘못된 개수의 인자를 넣었을 때 발생한다.
COUNT 함수는 하나의 컬럼 또는 값만 받을 수 있기 때문에 여러 개를 넣으면 오류가 발생한다.

세 번째는 SELECT가 중복된 경우이다.
"Expected end of input but got keyword SELECT"라는 메시지는 하나의 쿼리 안에서 SELECT가 여러 번 등장할 때 나타난다.
이는 쿼리를 구분하지 않고 이어서 작성했거나, 여러 쿼리를 한 번에 실행하려고 할 때 발생한다.
이 경우 세미콜론(;)으로 쿼리를 구분하거나, 실행할 쿼리만 선택해서 실행해야 한다.

네 번째는 WHERE 위치 오류이다.
"Expected end of input but got keyword WHERE"라는 메시지는 WHERE 절이 잘못된 위치에 있을 때 발생한다.
SQL에서는 각 구문의 순서가 정해져 있으며, WHERE는 FROM 뒤에 와야 하고 LIMIT보다 앞에 위치해야 한다.
순서가 틀리면 문법 오류가 발생한다.

정리하면, SQL 문법 오류를 해결하기 위해서는 에러 메시지를 정확히 읽고 의미를 이해하는 것이 가장 중요하다. 또한 함수의 사용 규칙과 SQL 구문의 순서를 올바르게 지키는 것이 필요하다.>



## 4-2. 데이터 타입과 데이터 변환(CAST, SAFE_CAST)

~~~
✅ 학습 목표 :
* 데이터 타입의 종류를 설명할 수 있다. 
* 데이터 타입을 변환하는 방법을 설명할 수 있다. 
~~~

<데이터 타입과 데이터 변환 (CAST, SAFE_CAST)

데이터를 다룰 때 가장 기본이 되는 개념 중 하나는 데이터 타입이다. 데이터 타입은 각 데이터가 어떤 형태를 가지는지를 정의하며, 올바른 타입을 사용하는 것은 정확한 데이터 처리와 분석을 위해 매우 중요하다.

먼저, 데이터 타입의 종류에는 여러 가지가 있다. 대표적으로 숫자를 저장하는 정수형(INT)과 실수형(FLOAT), 문자열을 저장하는 문자형(STRING), 날짜와 시간을 나타내는 DATE, DATETIME, TIMESTAMP 등이 있다. 이 외에도 TRUE/FALSE 값을 가지는 BOOLEAN 타입도 자주 사용된다. 각 타입은 저장 방식과 연산 방법이 다르기 때문에 상황에 맞게 적절한 타입을 선택해야 한다.

하지만 실제 데이터 분석 과정에서는 데이터 타입이 항상 원하는 형태로 주어지지 않는 경우가 많다. 예를 들어, 숫자가 문자열로 저장되어 있거나 날짜가 텍스트 형태로 들어오는 경우가 있다. 이런 상황에서는 데이터 타입을 변환해야 하며, 이를 위해 사용하는 것이 CAST 함수이다.

CAST는 특정 값을 원하는 데이터 타입으로 변환하는 함수이다. 예를 들어 문자열로 되어 있는 숫자를 정수로 바꾸고 싶다면 CAST를 사용하여 변환할 수 있다. CAST는 변환이 가능할 때는 정상적으로 수행되지만, 변환이 불가능한 경우에는 오류가 발생한다는 특징이 있다.

이러한 문제를 보완하기 위해 SAFE_CAST를 사용할 수 있다. SAFE_CAST는 CAST와 동일하게 데이터 타입을 변환하지만, 변환이 실패할 경우 오류를 발생시키는 대신 NULL 값을 반환한다. 따라서 데이터에 오류 가능성이 있을 때는 SAFE_CAST를 사용하는 것이 안정적인 방법이다.

정리하면, 데이터 타입은 데이터의 형태를 정의하는 중요한 요소이며, 상황에 따라 적절한 타입으로 변환하는 과정이 필요하다. CAST는 기본적인 타입 변환을 위한 함수이고, SAFE_CAST는 오류를 방지하며 안전하게 변환하기 위한 함수로, 데이터 처리 과정에서 매우 유용하게 활용된다.>



## 4-3. 문자열 함수(CONCAT, SPLIT, REPLACE, TRIM, UPPER)

~~~
✅ 학습 목표 :
* 문자열 함수들의 종류를 이해하고 어떠한 상황에서 사용하는지 설명할 수 있다. 
~~~

<문자열 함수 (CONCAT, SPLIT, REPLACE, TRIM, UPPER)

문자열 함수는 데이터 처리 과정에서 텍스트 데이터를 가공하고 원하는 형태로 변환할 때 사용하는 매우 중요한 기능이다. 특히 데이터 정제나 전처리 단계에서 자주 활용되며, 상황에 따라 적절한 함수를 선택하는 것이 중요하다.

먼저, CONCAT 함수는 여러 개의 문자열을 하나로 이어 붙일 때 사용한다. 예를 들어 이름과 성을 합쳐 하나의 컬럼으로 만들거나, 여러 텍스트 정보를 결합할 때 유용하다. 즉, 데이터를 연결하여 새로운 문자열을 만들고 싶을 때 사용하는 함수이다.

SPLIT 함수는 하나의 문자열을 특정 기준(구분자)을 기준으로 나누는 역할을 한다. 예를 들어 쉼표(,)나 공백을 기준으로 데이터를 나누어 각각의 요소로 분리할 때 사용한다. 주로 하나의 컬럼에 여러 값이 들어 있을 때 이를 분해하는 데 활용된다.

REPLACE 함수는 문자열 내의 특정 문자나 단어를 다른 값으로 바꾸고 싶을 때 사용한다. 예를 들어 특정 단어를 다른 표현으로 변경하거나, 불필요한 문자를 제거할 때 유용하다. 데이터 정제 과정에서 자주 쓰이는 함수이다.

TRIM 함수는 문자열의 앞뒤에 포함된 공백을 제거하는 함수이다. 데이터에 불필요한 공백이 포함되어 있으면 비교나 분석 시 오류가 발생할 수 있기 때문에, 이를 제거하는 데 사용된다. 특히 사용자 입력 데이터 처리 시 매우 중요하다.

UPPER 함수는 문자열을 모두 대문자로 변환하는 함수이다. 대소문자가 섞여 있는 데이터를 일관된 형태로 맞추고 싶을 때 사용한다. 예를 들어, 같은 단어라도 대소문자 차이로 인해 다르게 인식되는 문제를 방지할 수 있다.

정리하면, 문자열 함수는 데이터를 정리하고 원하는 형태로 가공하는 데 필수적인 도구이다. CONCAT은 문자열 결합, SPLIT은 분리, REPLACE는 치환, TRIM은 공백 제거, UPPER는 대문자 변환을 담당하며, 각각의 함수는 상황에 맞게 적절히 활용해야 한다.>



## 4-4. 날짜 및 시간 데이터 이해하기(1) (타임존, UTC, Millisecond, TIMESTAMP/DATETIME)

~~~
✅ 학습 목표 :
* 날짜 및 시간 데이터 타입과 UTC의 개념을 설명할 수 있다. 
* DATE, DATETIME, TIMESTAMP 에 대해서 설명할 수 있다.
* 시간함수들의 종류와 시간의 차이를 추출하는 방법을 설명할 수 있다. 
~~~

<날짜 및 시간 데이터 이해하기 (타임존, UTC, Millisecond, TIMESTAMP/DATETIME)

날짜와 시간 데이터는 데이터 분석에서 매우 중요한 요소이며, 특히 시간의 흐름이나 이벤트의 발생 시점을 분석할 때 필수적으로 사용된다. 이러한 데이터를 정확하게 다루기 위해서는 데이터 타입과 시간 개념에 대한 이해가 필요하다.

먼저, 날짜 및 시간 데이터 타입에는 DATE, DATETIME, TIMESTAMP 등이 있다. DATE는 연, 월, 일과 같은 날짜 정보만을 저장하는 타입이다. 시간 정보는 포함하지 않기 때문에 단순히 날짜 기준의 분석에 적합하다. DATETIME은 날짜와 시간을 함께 저장하는 타입으로, 특정 시점의 정확한 시간까지 표현할 수 있다. 다만, DATETIME은 타임존의 영향을 받지 않고 그대로 저장된 값을 유지한다는 특징이 있다. 반면 TIMESTAMP는 날짜와 시간을 저장하면서 동시에 타임존을 고려하는 타입이다. 즉, UTC 기준으로 저장되고, 조회할 때 각 지역의 시간으로 변환될 수 있다.

여기서 중요한 개념이 UTC(Coordinated Universal Time)이다. UTC는 전 세계에서 공통으로 사용하는 표준 시간으로, 각 나라의 시간은 UTC를 기준으로 계산된다. 예를 들어 한국은 UTC+9이므로 UTC 시간보다 9시간 빠르다. TIMESTAMP는 이러한 UTC 기준으로 데이터를 저장하기 때문에, 서로 다른 지역에서도 동일한 기준으로 시간을 비교할 수 있다는 장점이 있다.

또한 시간 데이터를 다룰 때 Millisecond(밀리초) 개념도 중요하다. 밀리초는 1초를 1,000으로 나눈 단위로, 매우 짧은 시간 간격을 표현할 때 사용된다. 로그 데이터나 실시간 처리 데이터에서는 밀리초 단위까지 정확하게 기록하는 경우가 많다.

시간 함수는 이러한 날짜 및 시간 데이터를 처리하고 분석하는 데 사용된다. 대표적으로 현재 시간을 가져오는 함수, 특정 부분(연도, 월, 일, 시 등)을 추출하는 함수, 그리고 시간 간의 차이를 계산하는 함수 등이 있다. 예를 들어, 두 날짜 사이의 차이를 계산할 때는 날짜 차이 함수(DATE_DIFF 등)를 사용할 수 있으며, 이를 통해 기간 분석이나 경과 시간 계산이 가능하다.

정리하면, DATE는 날짜만 저장하는 타입, DATETIME은 시간까지 포함하지만 타임존 영향을 받지 않는 타입, TIMESTAMP는 UTC 기반으로 저장되어 타임존을 고려하는 타입이다. 또한 UTC는 전 세계 공통 기준 시간이며, 시간 함수들을 활용하면 날짜와 시간 데이터를 효과적으로 분석할 수 있다.>

### 🎉 수고하셨습니다.
